Malicious VS Code Extensions Steal Source Code: 'MaliciousCorgi' Campaign

Over 100,000 developers unwittingly installed fake AI coding assistants that silently exfiltrated private GitHub repositories and AWS keys to servers in China.

TheHGTech Security Team January 30, 2026 7 min read Supply Chain
CAMPAIGN SUMMARY

Campaign Name: MaliciousCorgi
Affected Platform: VS Code Marketplace
Malware Behavior: Git repository scraping, AWS/Azure key theft.
Impact: 100k+ infected developer environments.
Status: Extensions removed by Microsoft, but persistent backdoors may remain.

Security researchers have uncovered a massive supply chain attack targeting the developer community. A cluster of malicious extensions in the Visual Studio Code Marketplace, collectively dubbed the "MaliciousCorgi" campaign, successfully infected over 100,000 developer machines by posing as AI coding assistants.

Unlike previous attacks that relied on typo-squatting, these extensions actually functioned as advertised—providing basic code completion—while silently executing a background process that scraped local data.

How The Attack Worked

1. The Lure: "AI Helper Pro"

The attackers published polished extensions with names like AI-Code-Helper-Pro and ChatMoss-GPT. They used fake reviews and even paid ads to boost their ranking in the Marketplace search results.

2. The Payload: Obfuscated WASM

To bypass Microsoft's security scans, the malicious logic was hidden inside a WebAssembly (WASM) module packaged with the extension. The JS code would call this WASM blob, which contained the compiled Go-based malware.

3. The Exfiltration

Once installed, the extension monitors file usage. When a developer opens a sensitive file (like .env, id_rsa, or config.json), the malware:

  • Reads the file content in memory.
  • Regex matches for AWS, Stripe, and OpenAI API keys.
  • Exfiltrates the data to a C2 server (api.maliciouscorgi.net) disguised as usage telemetry.

Indicators of Compromise (IoCs)

Malicious Extension IDs:
  • publisher.ai-code-helper-pro
  • publisher.chatmoss-gpt
  • tech.corgi-code-assist
C2 Domains:
  • telemetry-vscode-api.net
  • api.maliciouscorgi.net

What To Do If Infected

REMEDIATION STEPS
  1. Uninstall any extensions listed in the IoCs above immediately.
  2. Rotate Secrets: Assume all API keys and SSH keys present on the machine are compromised. Rotate them immediately.
  3. Scan for Persistence: Check for unexpected cron jobs or scheduled tasks, as newer variants establish persistence outside of VS Code.

This attack serves as a stark reminder: Just because an extension is in the official Marketplace does not mean it is safe. Always verify the publisher.
