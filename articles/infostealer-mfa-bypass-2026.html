Infostealer Malware: Why Your MFA Is Already Compromised

You deployed MFA. You trained users on phishing. You even implemented passwordless. None of it matters. Infostealer malware doesn't steal your credentials—it steals your authenticated session. By the time you read this, millions of corporate session tokens have already been harvested today.

Current Threat Level: Critical Published: Jan 27, 2026 21 min read Harish G

Active Threat

Infostealer malware operates as a Malware-as-a-Service (MaaS) ecosystem, with families like Lumma, RedLine, Vidar, and Raccoon harvesting an estimated 1+ million fresh credentials and session tokens daily. These stolen sessions are sold on dark web markets and Telegram channels within minutes of theft.

Key Point: Infostealers don't attempt to bypass MFA—they completely sidestep it by stealing already-authenticated sessions.

Executive Summary

Here's the uncomfortable truth security vendors won't put in their marketing: MFA doesn't protect against infostealers. Neither do passwordless solutions. Neither does "strong authentication."

Why? Because infostealers attack the layer after authentication. They don't care what security theatre you performed to prove your identity. Once your browser has a valid session cookie, that cookie can be stolen and replayed from anywhere in the world.

I've been tracking the infostealer ecosystem for the past year, and what I'm seeing is genuinely alarming. The scale of credential and session theft has reached industrial levels—and most organizations have zero visibility into whether their users' sessions are already for sale.

Key Takeaway: Infostealers represent a paradigm shift in credential theft. They attack the post-authentication layer, rendering traditional identity security controls ineffective. Session tokens are the new passwords.

How Infostealers Work

Infostealers are specialized malware designed for a single purpose: vacuum up everything of value from a victim's device and exfiltrate it before anyone notices. The infection-to-exfiltration cycle often completes in under 60 seconds.

Infection Vectors

  • Pirated Software: Cracked games, tools, and applications bundled with stealers—the #1 vector
  • YouTube/Social Media: "Free tool" links in video descriptions and comments
  • Malvertising: Fake download sites served through Google Ads
  • SEO Poisoning: Fake software download pages ranking in search results
  • Discord/Telegram: Direct messages with "screensavers," "game mods," or "free premium"
  • GitHub: Malicious repositories impersonating popular tools

What Gets Stolen

Browser Cookies (incl. session tokens)
Saved Passwords (all browsers)
Autofill Data (names, addresses, cards)
Cryptocurrency Wallets
Browser Extension Data (password managers)
Local Files (.txt, .doc, .pdf with keywords)
Screenshots
Keystrokes (some variants)
Gaming Platform Credentials (Steam, Epic, etc.)
FTP/SSH/Remote Access Credentials

The Attack Flow

1

Infection

User downloads and executes malicious file (often bypasses AV using packers, crypters, or signed binaries). Execution often requires no admin privileges.

2

Data Harvesting

Stealer immediately accesses browser data stores, reading SQLite databases for cookies, credentials, and autofill. Decrypts credentials using Windows DPAPI.

3

Token Extraction

Critically, steals active session cookies for major platforms: Microsoft 365, Google Workspace, Okta, AWS, GitHub, Slack, etc. These are already-authenticated sessions.

4

Exfiltration

Collected data ("logs") packaged and sent to attacker C2 server via HTTPS, Telegram bot, Discord webhook, or other channels. Entire process: <60 seconds.

5

Monetization

Logs are automatically sorted, parsed, and listed on dark web markets (Genesis, Russian Market) or Telegram channels. Enterprise logs command premium prices.

Why MFA Doesn't Help

This is the critical point most organizations miss. Let me explain exactly why every MFA implementation is vulnerable to infostealer attacks:

MFA Type Why It Fails Against Infostealers
SMS/Email OTP User already authenticated before infection; session cookie stolen post-MFA
Authenticator Apps (TOTP) Same issue—session cookie valid regardless of how MFA was performed
Push Notifications User approved push before infection; or attacker uses stolen session directly
Hardware Keys (FIDO2) Best protection, but still vulnerable if session cookie has long validity
Passwordless/Passkeys Session tokens still issued after authentication—same vulnerability
The Core Problem: Authentication happens once. The resulting session token is valid for hours, days, or weeks. Infostealers don't attack authentication—they steal the result of successful authentication.

Session Cookie Replay Attack

Here's what an attacker does with stolen session cookies:

  1. Purchase or receive "logs" from stealer operation
  2. Filter for high-value targets (corporate domains, specific services)
  3. Import cookies into browser using tool like EditThisCookie or custom script
  4. Navigate to target service—instantly logged in as victim
  5. No username, password, or MFA required

The attacker is now you, as far as the service is concerned. They have your existing, validated session. They can access your email, files, admin panels—anything your session token grants access to.

Major Infostealer Families

The infostealer ecosystem is dominated by several Malware-as-a-Service offerings:

Lumma Stealer MaaS

Currently dominant. $250-1000/month subscription. Excellent evasion, frequent updates.

Browsers, crypto, 2FA codes, Discord tokens

RedLine Stealer MaaS

Veteran player. $100-150 for lifetime license. Most documented family.

Browsers, FTP, VPN, Telegram, cryptocurrency

Vidar Stealer MaaS

Based on Arkei. Targets wide range of apps and platforms including gaming.

Browsers, 2FA, crypto wallets, gaming platforms

Raccoon Stealer MaaS

Returned as v2 after disruption. Easy deployment, reliable exfil.

Browsers, crypto, screenshot, system info

Stealc

Newer entrant. Strong targeting of browser-based password managers.

Browsers, password managers, crypto

RisePro

Focuses on browser extension data, particularly crypto and 2FA extensions.

Extensions, browsers, hardware fingerprints

Enterprise Impact

While infostealers often infect personal devices, the enterprise impact is severe:

The BYOD Problem

Employees access corporate services from personal devices. Those personal devices are used for gaming, downloading "free" tools, and high-risk browsing. Infostealer on personal device = corporate session tokens stolen.

Even if your corporate devices are locked down, if a user ever logged into M365 or Okta from their personal machine, their session could be compromised.

High-Value Targets by Sector

Enterprise IT

AWS/Azure/GCP console sessions, GitHub, CI/CD access

Financial

Evolution and Scale

2018-2019 Early Days: Infostealers primarily targeted gaming accounts, crypto wallets. Less sophisticated, smaller scale.
2020-2021 MaaS Explosion: RedLine, Raccoon, and others popularize subscription models. Dark web markets scale up.
2022 Enterprise Focus: Actors realize corporate sessions are more valuable. Targeting shifts to business users.
2023-2024 Anti-Detection Arms Race: Lumma and others invest heavily in evasion. Polymorphic loaders, signed binaries.
2025 Industrial Scale: Genesis Market takedown briefly disrupts, but ecosystem rapidly recovers and expands.
2026 Current State: Estimated 1M+ fresh logs daily. Automated sorting, instant monetization. Enterprise sessions as commodity.

Defense Strategies

Traditional security controls have limited effectiveness against infostealers. Here's what actually works:

Most Effective Controls

Recommended Defenses:
  • Hardware Security Keys (FIDO2) + Short Session Duration: Best combination—even if session stolen, it expires quickly
  • Device Trust/Certificate-Based Access: Require known, managed devices for access to sensitive systems
  • Session Token Binding: Bind tokens to device fingerprints; reject replayed tokens from different devices
  • Browser Isolation: Enterprise browsing in isolated containers; session tokens never touch local device
  • Continuous Session Monitoring: Detect impossible travel, new device fingerprints mid-session
  • Credential Monitoring: Services that scan dark web markets for your domain's compromised credentials

Detection Opportunities

Detection Point What to Look For
Session Analytics Same session token from different IPs, user agents, or locations
Endpoint Detection Credential access, browser data file reads, unusual DPAPI calls
Dark Web Monitoring Your corporate domain appearing in stealer log markets
Authentication Logs Token reuse from new device/location without re-authentication
User Behavior Actions inconsistent with user's normal patterns post-session hijack

Technical Controls

# Example: Azure AD Conditional Access for Token Protection
# Requires tokens to be bound to the signing device

New-AzureADMSConditionalAccessPolicy -DisplayName "Require token binding" `
  -State "Enabled" `
  -Conditions @{
    Applications = @{ IncludeApplications = "All" }
    Users = @{ IncludeUsers = "All" }
  } `
  -SessionControls @{
    SignInFrequency = @{ Value = 4; Type = "hours" }
    PersistentBrowser = @{ Mode = "never" }
  }

User Education

While limited in effectiveness, users should understand:

  • Pirated software is the #1 infection vector—not worth the risk
  • Corporate accounts on personal devices = corporate risk
  • Signs of infostealer infection: unusual browser behavior, new extensions, antivirus alerts
  • If potentially infected, report immediately for password reset and session revocation

The Visibility Gap

Here's what concerns me most: most organizations have no idea if they're already compromised.

Infostealer infections often:

  • Don't trigger endpoint protection (sophisticated evasion)
  • Leave no artifacts (memory-only or self-deleting execution)
  • Complete before any detection is possible (sub-60-second operation)

The first indication of compromise is often:

  • Account takeover detected after damage is done
  • Your credentials appearing on a dark web monitoring alert
  • Ransomware deployment weeks/months later using stolen access
The Bottom Line: Infostealers have fundamentally broken the assumption that MFA protects accounts. Session tokens are the new credentials, and they're being harvested at industrial scale. Organizations need to move beyond authentication-focused security to session-level protection: token binding, continuous validation, and aggressive expiration. The "trust but verify" model of session management is obsolete.
