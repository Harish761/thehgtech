Microsoft Patch Tuesday January 2026: 114 CVEs, 3 Zero-Days, and One Actively Exploited

Microsoft kicks off 2026 with a massive security update addressing 114 vulnerabilities including CVE-2026-20805, an information disclosure flaw in Windows DWM that attackers are already weaponizing in the wild.

Released: January 14, 2026 Published: Jan 16, 2026 20 min read By Harish G
114 Total CVEs Patched
3 Zero-Day Vulnerabilities
8 Critical Severity
1 Actively Exploited

Executive Summary

Microsoft's first Patch Tuesday of 2026 is one for the record books. Released on January 14, 2026, this security update addresses 114 vulnerabilities across the Windows ecosystem, including three zero-day vulnerabilities—one of which is confirmed to be actively exploited in the wild.

The star of this month's update is CVE-2026-20805, an information disclosure vulnerability in Windows Desktop Window Manager (DWM) that threat actors are actively leveraging as part of exploit chains. Despite its moderate CVSS score of 5.5, its active exploitation status means security teams should treat this as a priority-one patch.

Beyond the zero-days, this update includes eight critical-severity vulnerabilities affecting Microsoft Office, Windows LSASS, Windows NTFS, and the Graphics Component. Several of these enable remote code execution (RCE) through maliciously crafted files—a classic attack vector that continues to plague enterprise environments.

Urgent Action Required: CVE-2026-20805 is actively exploited in the wild and has been added to CISA's Known Exploited Vulnerabilities (KEV) catalog. Federal agencies must remediate by February 3, 2026. All organizations should patch immediately.

The Actively Exploited Zero-Day: CVE-2026-20805

CVE-2026-20805 CVSS 5.5 ACTIVELY EXPLOITED

Windows Desktop Window Manager (DWM) Information Disclosure Vulnerability

What is CVE-2026-20805?

CVE-2026-20805 is an information disclosure vulnerability in the Windows Desktop Window Manager (DWM), the core Windows component responsible for desktop composition and visual effects. The vulnerability allows an attacker to leak kernel memory addresses—information that shouldn't be accessible to regular applications.

You might wonder why a "mere" information disclosure bug warrants critical attention. The answer lies in how modern exploitation works. Attackers rarely use a single vulnerability—they chain multiple bugs together. Memory address leaks like CVE-2026-20805 are invaluable for bypassing Address Space Layout Randomization (ASLR), one of Windows' fundamental security defenses.

The Attack Chain

Security researchers have observed CVE-2026-20805 being used in a multi-stage attack chain:

Stage 1: Initial Access Attacker delivers malicious document or tricks user into visiting weaponized website
Stage 2: Memory Leak (CVE-2026-20805) Vulnerability exploited to leak kernel memory addresses, defeating ASLR
Stage 3: Privilege Escalation Leaked addresses used to precisely target a kernel vulnerability for SYSTEM privileges
Stage 4: Persistence Attacker establishes persistent access, deploys malware or ransomware

Who is Exploiting CVE-2026-20805?

Microsoft has not publicly attributed the exploitation to a specific threat actor, but intelligence sources suggest the vulnerability is being used by both state-sponsored groups and financially motivated attackers. The broad utility of ASLR bypass techniques makes this vulnerability valuable to virtually any advanced threat actor.

What's particularly concerning is the speed of exploitation. The vulnerability was being exploited before the patch was released (hence "zero-day"), indicating either:

  • Independent discovery by threat actors
  • Possible leak of vulnerability details
  • Sophisticated research capabilities of the attacking group

The Other Zero-Days

While CVE-2026-20805 gets the headlines for active exploitation, Microsoft patched two additional zero-day vulnerabilities that were publicly disclosed before the patch:

CVE-2023-31096: Agere Soft Modem Driver Privilege Escalation

Property Value
CVE ID CVE-2023-31096
Type Elevation of Privilege
Component Windows Agere Soft Modem Driver (agrsm64.sys / agrsm.sys)
Resolution Affected drivers removed from Windows

This is an interesting case—the CVE was originally assigned by MITRE back in 2023, but it took until 2026 for Microsoft to address it. The delay is due to the affected drivers being third-party components (LSI/Agere) that were bundled with Windows. Microsoft's resolution is definitive: they've simply removed the vulnerable drivers from the operating system entirely.

CVE-2026-21265: Secure Boot Security Feature Bypass

Property Value
CVE ID CVE-2026-21265
CVSS Score 6.4 (Medium)
Type Security Feature Bypass
Attack Vector Local, requires high privileges

CVE-2026-21265 is a Secure Boot bypass that could allow an authenticated attacker with high privileges to circumvent security features through weaknesses in the certificate update mechanism. While it requires local access and elevated privileges, Secure Boot bypasses are particularly valuable for attackers seeking to establish persistent, boot-level malware that survives system reinstallation.

Key Takeaway: Secure Boot vulnerabilities are rare but impactful. If you're running environments where firmware integrity is critical (financial systems, critical infrastructure), prioritize this patch even though it's not rated critical.

Critical Vulnerabilities: The Remote Code Execution Threats

Microsoft rated eight vulnerabilities as Critical in this release. Here are the ones security teams should focus on:

Microsoft Office RCE Vulnerabilities

CVE ID Component CVSS Preview Pane?
CVE-2026-20944 Microsoft Word 8.4 No
CVE-2026-20955 Microsoft Excel 7.8 No
CVE-2026-20957 Microsoft Excel 7.8 No
CVE-2026-20952 Microsoft Office Critical Yes!
CVE-2026-20953 Microsoft Office Critical Yes!

The most dangerous of these are CVE-2026-20952 and CVE-2026-20953. These Office vulnerabilities can be triggered through Outlook's Preview Pane—meaning a user doesn't even need to open the malicious attachment. Simply selecting the email in Outlook is enough to trigger exploitation.

Critical Risk: CVE-2026-20952 and CVE-2026-20953 can be exploited through Outlook Preview Pane. Disable the Preview Pane immediately as a temporary mitigation until patches are deployed.

Windows LSASS Remote Code Execution

CVE ID CVE-2026-20854
Component Windows Local Security Authority Subsystem Service (LSASS)
CVSS Score 7.5
Attack Vector Network

LSASS is the heart of Windows authentication. It handles login validation, password changes, and security token creation. A remote code execution vulnerability in LSASS is particularly dangerous because:

  • LSASS runs with SYSTEM privileges—the highest privilege level in Windows
  • Successful exploitation gives attackers complete control over the target
  • LSASS is network-accessible in domain environments
  • Domain controllers running LSASS are high-value targets

Windows NTFS Heap Buffer Overflow

CVE-2026-20840 and CVE-2026-20922 are heap-based buffer overflow vulnerabilities in Windows NTFS, the primary filesystem for Windows. Both have a CVSS score of 7.8 and enable RCE when processing specially crafted disk images or mounted volumes.

Attack scenarios include:

  • User mounts a malicious USB drive
  • User extracts a malicious ISO or VHD file
  • Attacker tricks user into mounting a network share containing exploit

Remediation and Mitigation Strategies

Immediate Actions (Within 24-48 Hours)

Priority 1 - Do Now:
  • Patch CVE-2026-20805 on all Windows endpoints immediately—it's actively exploited
  • Disable Outlook Preview Pane to mitigate CVE-2026-20952/20953 until patches are deployed
  • Prioritize Domain Controllers for the LSASS vulnerability (CVE-2026-20854)
  • Block untrusted USB devices to mitigate NTFS vulnerabilities while patching

Standard Patching Timeline

  • Critical/Actively Exploited: 24-48 hours
  • Critical RCE: 7 days
  • High Severity: 14 days
  • Medium/Low: 30 days (standard patch cycle)

Detection and Hunting

While patching, security teams should hunt for signs of exploitation:

  • CVE-2026-20805: Monitor for unusual DWM.exe behavior, unexpected memory reads, or process injection attempts targeting dwm.exe
  • Office exploits: Watch for suspicious Office process spawning child processes (cmd.exe, powershell.exe, mshta.exe)
  • LSASS attacks: Enable Credential Guard if not already active; monitor LSASS for unusual network connections

Impact on Different Industries

Healthcare

Healthcare organizations should prioritize patching immediately. With the ransomware epidemic continuing to target hospitals, the combination of Office RCE vulnerabilities and the actively exploited DWM bug creates significant risk for clinical systems.

Financial Services

The Secure Boot bypass (CVE-2026-21265) is particularly relevant for financial institutions where boot integrity is a compliance requirement. Domain controllers running LSASS are also high-value targets that require immediate attention.

Critical Infrastructure

OT environments often have delayed patching cycles, but the active exploitation of CVE-2026-20805 means infrastructure operators should accelerate testing and deployment. Network segmentation between IT and OT remains critical.

Education

Educational institutions are frequent targets for ransomware. The Outlook Preview Pane vulnerabilities are particularly dangerous in environments where users may click on anything. Disable Preview Pane institution-wide as an immediate mitigation.

Future Implications

The Exploit Chain Problem

CVE-2026-20805 illustrates a growing trend: attackers use multiple vulnerabilities together. A "moderate" information disclosure bug becomes critical when it enables ASLR bypass for a kernel exploit. Security teams need to evaluate vulnerabilities not just by CVSS score, but by their utility in attack chains.

Zero-Day Velocity

Three zero-days in one Patch Tuesday continues the trend of increased zero-day exploitation. Microsoft's own data shows zero-day exploitation has increased 50% year-over-year. Organizations need:

  • Faster patching capabilities
  • Better exploit detection (behavioral, not just signature-based)
  • Reduce attack surface through principles like least privilege

The Preview Pane Continues to Haunt

Outlook's Preview Pane has been a source of zero-click vulnerabilities for years. CVE-2026-20952 and CVE-2026-20953 add to this legacy. Organizations should seriously consider disabling the Preview Pane by default—the convenience isn't worth the recurring security risk.

Key Takeaways

Summary:
  1. Patch CVE-2026-20805 immediately—it's actively exploited and added to CISA KEV
  2. Disable Outlook Preview Pane to mitigate two critical Office RCE bugs
  3. Prioritize Domain Controllers for the LSASS vulnerability
  4. Block untrusted removable media while patching NTFS vulnerabilities
  5. 114 total vulnerabilities means thorough testing is essential—but don't let that delay critical patches
  6. Hunt for exploitation of the actively exploited zero-day in your environment
