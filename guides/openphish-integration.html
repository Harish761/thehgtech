OpenPhish Feed Integration Guide

ðŸ“– 8 min read Advanced ðŸ“… Updated: November 2025

Phishing remains one of the most effective attack vectors, with over 90% of cyberattacks starting with a phishing email. OpenPhish provides a free, real-time feed of active phishing URLs that you can integrate into your security infrastructure to detect and block phishing attempts before they reach your users.

What is OpenPhish?

OpenPhish is a fully automated platform that identifies phishing sites using proprietary Artificial Intelligence algorithms. The service provides:

  • Real-time phishing URL feed - Updated continuously with newly discovered phishing sites
  • Free tier - Public feed with basic phishing URLs
  • Premium tier - Enhanced feed with additional metadata and faster updates
  • JSON and text formats - Easy integration into various security tools

Why Integrate OpenPhish?

Proactive Defense

Block phishing URLs before they reach your users' inboxes. Email gateways, web proxies, and DNS filters can use OpenPhish data to prevent access to known phishing sites.

Incident Response

When investigating suspicious emails, quickly check if URLs are in the OpenPhish database. This helps triage potential phishing incidents faster.

Threat Hunting

Search your proxy logs, email logs, and DNS logs for historical matches against OpenPhish URLs to identify users who may have been targeted.

Understanding the OpenPhish Feed

Feed Format

OpenPhish provides feeds in two formats:

1. Text Format (Free Tier)

https://example-phishing-site.com/login
https://another-phishing-page.net/verify
https://fake-bank-login.com/account

Simple newline-separated list of phishing URLs. Easy to parse and integrate.

2. JSON Format (Premium Tier)

{
  "url": "https://example-phishing-site.com/login",
  "phish_detail_url": "https://openphish.com/phish/...",
  "submission_time": "2025-11-26T12:34:56Z",
  "verified": "yes",
  "target": "PayPal",
  "sector": "Financial Services"
}

Enhanced format with metadata including target brand, verification status, and submission time.

Automation Script Example

Here's a complete Python script to fetch, parse, and integrate OpenPhish data: 

#!/usr/bin/env python3
"""
OpenPhish Feed Integration Script
Fetches OpenPhish feed and integrates with security tools
"""

import requests
import json
import sqlite3
from datetime import datetime
from urllib.parse import urlparse

class OpenPhishIntegration:
    def __init__(self, db_path='openphish.db'):
        self.feed_url = 'https://openphish.com/feed.txt'
        self.db_path = db_path
        self.init_database()
    
    def init_database(self):
        """Initialize SQLite database for storing phishing URLs"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute('''
            CREATE TABLE IF NOT EXISTS phishing_urls (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                url TEXT UNIQUE,
                domain TEXT,
                first_seen TIMESTAMP,
                last_seen TIMESTAMP,
                status TEXT DEFAULT 'active'
            )
        ''')
        conn.commit()
        conn.close()
    
    def fetch_feed(self):
        """Fetch latest OpenPhish feed"""
        try:
            response = requests.get(self.feed_url, timeout=30)
            response.raise_for_status()
            return response.text.strip().split('\n')
        except requests.exceptions.RequestException as e:
            print(f"Error fetching feed: {e}")
            return []
    
    def parse_url(self, url):
        """Extract domain from URL"""
        parsed = urlparse(url)
        return parsed.netloc
    
    def update_database(self, urls):
        """Update database with new phishing URLs"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        now = datetime.now()
        
        for url in urls:
            domain = self.parse_url(url)
            cursor.execute('''
                INSERT INTO phishing_urls (url, domain, first_seen, last_seen)
                VALUES (?, ?, ?, ?)
                ON CONFLICT(url) DO UPDATE SET last_seen = ?
            ''', (url, domain, now, now, now))
        
        conn.commit()
        conn.close()
        print(f"Updated database with {len(urls)} URLs")
    
    def get_active_domains(self):
        """Get list of active phishing domains"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute('''
            SELECT DISTINCT domain FROM phishing_urls
            WHERE status = 'active'
        ''')
        domains = [row[0] for row in cursor.fetchall()]
        conn.close()
        return domains
    
    def check_url(self, url):
        """Check if URL is in phishing database"""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute('''
            SELECT url, first_seen FROM phishing_urls
            WHERE url = ? AND status = 'active'
        ''', (url,))
        result = cursor.fetchone()
        conn.close()
        return result is not None
    
    def run(self):
        """Main execution"""
        print("Fetching OpenPhish feed...")
        urls = self.fetch_feed()
        
        if urls:
            print(f"Retrieved {len(urls)} phishing URLs")
            self.update_database(urls)
            
            # Export for other tools
            domains = self.get_active_domains()
            with open('phishing_domains.txt', 'w') as f:
                f.write('\n'.join(domains))
            print(f"Exported {len(domains)} unique domains")
        else:
            print("No URLs retrieved")

if __name__ == '__main__':
    integrator = OpenPhishIntegration()
    integrator.run()

Best Practices

  • Update frequently - Fetch the feed every 30-60 minutes for fresh data
  • Implement TTL - Remove URLs older than 7 days (phishing sites are often short-lived)
  • Validate before blocking - Test URLs before adding to production blocklists
  • Monitor false positives - Track blocked URLs and investigate user complaints
  • Combine with other feeds - Use OpenPhish alongside PhishTank, URLhaus, etc.
  • Log everything - Keep records of blocked attempts for incident response

Common Pitfalls

1. Blocking Legitimate Sites

Problem: Occasionally, legitimate sites may be incorrectly flagged.

Solution: Implement a whitelist for known-good domains and provide users with a way to report false positives.

2. Stale Data

Phishing sites are often taken down quickly. Blocking a URL that no longer exists wastes resources.

  • Implement automatic expiration for old URLs
  • Periodically verify if URLs are still active
  • Remove inactive URLs from blocklists

3. Performance Impact

Large blocklists can impact firewall/proxy performance.

  • Limit blocklist size (e.g., 10,000 most recent URLs)
  • Use domain-level blocking instead of full URLs when possible
  • Optimize database queries for fast lookups

Measuring Effectiveness

Track these metrics to evaluate your OpenPhish integration:

  • Blocked Attempts - How many phishing attempts were blocked?
  • User Reports - How many users reported blocked phishing emails?
  • False Positives - How many legitimate sites were incorrectly blocked?
  • Coverage - What percentage of phishing attempts are caught?
  • Response Time - How quickly are new phishing URLs blocked?

Advanced: Premium Feed Features

The OpenPhish premium feed provides additional metadata:

  • Target Brand - Which company/service is being impersonated
  • Verification Status - Confidence level of phishing classification
  • Sector Information - Industry being targeted
  • Faster Updates - More frequent feed refreshes
  • Historical Data - Access to past phishing campaigns
