1. What is MITRE ATT&CK?

Understanding the Basics

Imagine you're learning to defend a castle. You'd want to know:

  • How do attackers typically get in? (scaling walls, breaking gates, bribery...)
  • What do they do once inside? (search for treasure, take hostages, burn buildings...)
  • How have past attackers operated? (specific techniques used by famous armies)

MITRE ATT&CK is exactly this - but for cyber attacks. It's a comprehensive "playbook" of how hackers actually attack organizations, based on real incidents that have been studied and documented over years.

Think of it like this: If hackers were chess players, ATT&CK would be a book containing EVERY possible move they might make, organized by what they're trying to achieve. Instead of just knowing "hackers are bad," you know specifically: "Hackers often use PowerShell (T1059.001) to run commands because it's already installed on Windows computers."

The Technical Definition

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) is a globally-accessible knowledge base of adversary tactics and techniques based on real-world observations. It serves as a common language for describing cyber attacks.

The framework is maintained by MITRE, a nonprofit that operates federally funded R&D centers. They've catalogued thousands of attacks and distilled them into a structured framework anyone can use.

Why ATT&CK Changed Everything

Before ATT&CK, security teams had a problem. When analysts discussed attacks, they'd say things like:

  • "The attacker ran some scripts"
  • "They moved around the network"
  • "They stole data somehow"

This was vague and made it hard to compare attacks, build defenses, or communicate clearly.

Now, with ATT&CK, those same descriptions become:

  • "T1059.001 - PowerShell execution"
  • "T1021.001 - Remote Desktop Protocol lateral movement"
  • "T1048 - Exfiltration over alternative protocol"
Real-World Example: When the Colonial Pipeline ransomware attack happened in 2021, security teams worldwide immediately shared analysis mapped to ATT&CK. Within hours, organizations could check: "Do we detect T1078 (Valid Accounts)? Do we detect T1486 (Data Encrypted for Impact)?" This shared language accelerated global defense.

2. Framework Structure

Understanding the Hierarchy (It's Like an Outline)

ATT&CK is organized like a well-structured document with increasing levels of detail:

Level What It Is Example Count
Matrices The environment (Windows/Mac/Linux, Mobile, Industrial) Enterprise Matrix 3
Tactics The adversary's GOAL (the "WHY") Credential Access 14
Techniques HOW they achieve the goal OS Credential Dumping ~200
Sub-techniques Specific variations of techniques LSASS Memory ~400
Procedures Exact implementation by specific threat actors Mimikatz sekurlsa::logonpasswords Thousands
Walk-Through Example:
Tactic: Credential Access (goal: steal credentials)
Technique: T1003 - OS Credential Dumping
Sub-technique: T1003.001 - LSASS Memory
Procedure: APT28 uses Mimikatz sekurlsa::logonpasswords command

Translation: The attacker wanted passwords (tactic), so they dumped operating system credentials (technique), specifically from LSASS memory (sub-technique), using the exact Mimikatz command the notorious APT28 group uses (procedure).

The Three Matrices

  • Enterprise: Windows, macOS, Linux, Cloud (AWS/Azure/GCP), Network, Containers - this is the most commonly used matrix
  • Mobile: iOS and Android device attacks
  • ICS: Industrial Control Systems (power plants, manufacturing, SCADA)
Common Beginner Mistake: Don't try to memorize all 200+ techniques! Start with the 10-15 most common techniques in your industry (check reports from CrowdStrike, Mandiant, or CISA). You'll naturally learn more as you encounter them in incidents.

3. The 14 Tactics Explained

Tactics represent the adversary's objective at each phase of an attack:

TA0043: Reconnaissance

Gather information to plan attack

TA0042: Resource Development

Establish attack infrastructure

TA0001: Initial Access

Get into the network

TA0002: Execution

Run malicious code

TA0003: Persistence

Maintain foothold

TA0004: Privilege Escalation

Gain higher permissions

TA0005: Defense Evasion

Avoid detection

TA0006: Credential Access

Steal credentials

TA0007: Discovery

Learn the environment

TA0008: Lateral Movement

Move through network

TA0009: Collection

Gather target data

TA0011: Command & Control

Communicate with implants

TA0010: Exfiltration

Steal data out

TA0040: Impact

Disrupt, destroy, manipulate

4. Understanding Techniques

Technique Components

Field Description
ID Unique identifier (e.g., T1059)
Name Descriptive name (e.g., Command and Scripting Interpreter)
Description How the technique works
Platforms Where it applies (Windows, Linux, macOS, etc.)
Data Sources Logs/telemetry for detection
Detection Guidance on detecting the technique
Mitigations How to prevent/reduce impact

Most Common Techniques (2024-2025)

  • T1059 - Command and Scripting Interpreter (PowerShell, Python)
  • T1078 - Valid Accounts (compromised credentials)
  • T1566 - Phishing (spearphishing attachments/links)
  • T1486 - Data Encrypted for Impact (ransomware)
  • T1082 - System Information Discovery

5. How to Map Threats

Mapping Incident to ATT&CK

  1. Extract Behaviors: What did the adversary DO? (not just what tools)
  2. Identify Tactics: What was their goal at each step?
  3. Match Techniques: Find the technique that describes the behavior
  4. Document Evidence: Link to logs/artifacts that prove the technique

Example Mapping

Observed Behavior Tactic Technique
User clicked phishing link Initial Access T1566.002 Spearphishing Link
Macro executed PowerShell Execution T1059.001 PowerShell
Added to registry Run key Persistence T1547.001 Registry Run Keys
Dumped LSASS memory Credential Access T1003.001 LSASS Memory
RDP to other servers Lateral Movement T1021.001 Remote Desktop

6. Detection Engineering

Building ATT&CK-Based Detections

  1. Choose Technique: Start with high-priority techniques based on threat landscape
  2. Review Data Sources: What logs do you need?
  3. Build Detection Logic: Create SIEM rule or query
  4. Test with Emulation: Use Atomic Red Team or similar
  5. Tune for Environment: Reduce false positives
  6. Document Coverage: Map detection to technique ID

Example Detection (Splunk)

# T1003.001 - LSASS Memory Access
index=sysmon EventCode=10 TargetImage="*lsass.exe"
| where NOT match(SourceImage, "(?i)(MsMpEng|csrss|wininit)")
| stats count by SourceImage, SourceUser, Computer
| where count > 0

7. Coverage Analysis

Use ATT&CK to identify detection gaps:

Coverage Assessment Steps

  1. Inventory all current detection rules
  2. Map each rule to ATT&CK technique(s)
  3. Visualize coverage using ATT&CK Navigator
  4. Identify gaps in critical tactics
  5. Prioritize based on threat intelligence
Common Gaps: Organizations often have strong coverage for Initial Access and Execution, but poor coverage for Defense Evasion and Credential Access. Focus detection efforts on these blind spots.

8. ATT&CK Navigator

The ATT&CK Navigator is a web tool for visualizing and annotating the ATT&CK matrix.

Navigator Use Cases

  • Visualize detection coverage (green = detected, red = gap)
  • Compare threat actor TTPs to your defenses
  • Plan red team operations
  • Track remediation progress
  • Create layers for different analysis views

Access Navigator: mitre-attack.github.io/attack-navigator
