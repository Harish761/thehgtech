1. What is Threat Intelligence?

Understanding the Basics

Imagine you're a store detective trying to catch shoplifters. It would help to know what past shoplifters looked like, what tactics they used, and which stores they hit recently. That's essentially what threat intelligence is for cybersecurity.

Threat intelligence feeds are like a constantly updated "watch list" that tells your security tools:

  • These IP addresses are known to be malicious (blocklist them)
  • These website domains are running phishing scams (block or alert)
  • Files with these signatures are malware (stop them from running)
  • This is how the latest attack campaigns are operating (watch for this behavior)
Think of it like this: If your firewall and antivirus are your security guards, threat intelligence is like giving them a constantly updated "FBI Most Wanted" list. Instead of just reacting to bad behavior, they can proactively block known bad actors before they cause damage.

Why Threat Intelligence Matters

Without Threat Intel With Threat Intel
Your firewall only blocks what it's explicitly configured to block Firewall automatically blocks thousands of known malicious IPs
Antivirus relies only on signatures from that vendor New malware hashes pushed from multiple sources within hours
Phishing detection is limited to major campaigns Real-time phishing URL feeds block new threats immediately
You find out about attacks after you're hit Proactive detection: "This IP attacked another company yesterday"

What is an IOC? (Indicator of Compromise)

IOCs are the "fingerprints" of malicious activity. When security researchers analyze attacks, they document observable patterns that can identify the same attack elsewhere:

  • IP Address: The attacker's server (e.g., 185.141.27.xxx)
  • Domain: Malicious website (e.g., fake-update.com)
  • File Hash: Unique fingerprint of malware (SHA256)
  • URL: Specific malicious page (e.g., phishing login page)
Common Misconception: Threat intelligence is NOT just about blocking. Good threat intel also enriches your investigations. When you see a suspicious IP in your logs, threat intel can tell you: "This IP is associated with APT28, a Russian state-sponsored group known for targeting government organizations."

The Technical Definition

Threat Intelligence (TI) is evidence-based knowledge about existing or emerging threats that helps organizations prevent, detect, and respond to cyber attacks. It ranges from tactical IOCs to strategic threat assessments.

Intelligence Levels

Level Description Audience Example
Strategic High-level trends, threat actors, motivations Executives, CISO "China-based attacks on semiconductor companies up 40%"
Operational TTPs, campaigns, threat actor profiles Security managers "APT41 is targeting VPN vulnerabilities with this campaign"
Tactical IOCs: IPs, domains, hashes, URLs SOC analysts, SIEM "Block these 500 C2 domains immediately"
Technical Malware analysis, exploit details Malware analysts, IR "This malware uses DLL sideloading via this technique"
Where to Start: If you're new to threat intelligence, start with 3-5 free feeds (abuse.ch, AlienVault OTX, OpenPhish). Import them as lookup tables in your SIEM and create alerts when your logs match known-bad indicators. You'll be amazed what you find.

4. STIX/TAXII Standards

STIX and TAXII are OASIS standards for sharing threat intelligence:

STIX (Structured Threat Information eXpression)

A language and serialization format for threat intelligence objects:

{
  "type": "indicator",
  "id": "indicator--a932fcc6-e032-176c-126f-cb970a5a1fff",
  "pattern": "[domain-name:value = 'malicious.com']",
  "pattern_type": "stix",
  "valid_from": "2025-01-01T00:00:00Z",
  "labels": ["malicious-activity"]
}

TAXII (Trusted Automated eXchange of Intelligence Information)

A protocol for exchanging STIX data. TAXII 2.1 uses REST APIs.

  • Collections: Groups of intelligence objects
  • Channels: Push-based intelligence sharing
  • Discovery: Find available TAXII services

Splunk Integration Example

# Create lookup table from threat feed
| inputlookup threat_ips.csv
| eval threat_type="malicious_ip"

# Correlate with logs
index=firewall 
| lookup threat_ips.csv ip as dest_ip OUTPUT threat_type
| where isnotnull(threat_type)

7. Operational Workflow

Threat Intel Operational Cycle

  1. Collect: Ingest feeds from multiple sources
  2. Normalize: Convert to common format (STIX)
  3. Enrich: Add context (WHOIS, geolocation, related IOCs)
  4. Analyze: Correlate and prioritize
  5. Disseminate: Push to detection tools (SIEM, FW, EDR)
  6. Act: Alert, block, or investigate matches
  7. Feedback: Improve based on false positive rates

8. Feed Quality & Confidence

Quality Matters: Not all threat feeds are equal. Poor quality feeds lead to alert fatigue and false positives. Always assign confidence levels and track feed performance.

Confidence Scoring

  • High (80-100%): Verified by multiple sources, recent, active
  • Medium (40-79%): Single source, reasonable age, some validation
  • Low (0-39%): Old data, unverified, high false positive rate

Feed Hygiene

  • Set expiration dates on IOCs (30-90 days typical)
  • Track false positive rates per feed
  • Deduplicate across feeds
  • Retire underperforming feeds

Automated Enrichment

Use SOAR platforms to automate threat intel workflows:

  • Auto-enrich IOCs on detection
  • Auto-block confirmed malicious indicators
  • Auto-create tickets for investigation
  • Auto-update feeds and blocklists

  3. Build correlation rules for IOC matches
  4. Track false positive rates
  5. Consider MISP or OpenCTI if scaling
