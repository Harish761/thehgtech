DevSecOps Pipeline Security

DevOps

Integrate security into CI/CD pipelines with SAST, DAST, SCA, and shift-left practices.

20 min read
Intermediate
Jan 17, 2026

Traditional security testing at the end of development is too slow and too expensive. DevSecOps shifts security left - integrating automated security testing throughout the CI/CD pipeline so vulnerabilities are found when they're cheapest to fix.

Why Shift-Left Security?

A vulnerability found in development costs $80 to fix. Found in testing: $240. Found in production: $7,600. The earlier you find it, the cheaper it is. (Source: IBM Systems Sciences Institute)

1. DevSecOps Overview

The DevSecOps Pipeline

Security checks at every stage:

Stage Security Activity Tools
Code Pre-commit hooks, secrets scanning GitLeaks, pre-commit
Build SAST, SCA, license compliance Semgrep, Snyk, FOSSA
Test DAST, API security testing OWASP ZAP, Nuclei
Deploy Container scanning, IaC scanning Trivy, Checkov
Production RASP, WAF, monitoring Datadog, Falco

2. SAST - Static Application Security Testing

SAST analyzes source code to find vulnerabilities without running the application.

Popular SAST Tools

Tool Languages Highlights
Semgrep 30+ languages Fast, open-source, custom rules
SonarQube 25+ languages Quality + security, self-hosted
CodeQL 10+ languages GitHub native, query-based
Checkmarx 25+ languages Enterprise, accurate

GitHub Actions: Semgrep Example

.github/workflows/sast.yml
name: SAST Scan
on: [push, pull_request]

jobs:
  semgrep:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run Semgrep
        uses: returntocorp/semgrep-action@v1
        with:
          config: p/security-audit p/owasp-top-ten
        env:
          SEMGREP_APP_TOKEN: ${{ secrets.SEMGREP_APP_TOKEN }}

3. SCA - Software Composition Analysis

SCA scans dependencies for known vulnerabilities (CVEs) and license issues.

Supply Chain Risk

Log4Shell (CVE-2021-44228) showed how a single vulnerable dependency can affect thousands of applications. SCA is critical for modern software.

Popular SCA Tools

Tool Ecosystems Highlights
Snyk npm, Maven, pip, Go, etc. Fix PRs, container scanning
Dependabot GitHub native Free, auto PR updates
OWASP Dependency-Check Java, .NET, npm, etc. Open-source, offline
Renovate All major Highly configurable

GitHub Actions: Snyk Example

SCA with Snyk
- name: Run Snyk
  uses: snyk/actions/node@master
  env:
    SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
  with:
    args: --severity-threshold=high

4. Secrets Detection

Prevent API keys, passwords, and tokens from reaching your repository.

Pre-Commit Secrets Scanning

.pre-commit-config.yaml
repos:
  - repo: https://github.com/gitleaks/gitleaks
    rev: v8.18.0
    hooks:
      - id: gitleaks
        
  - repo: https://github.com/Yelp/detect-secrets
    rev: v1.4.0
    hooks:
      - id: detect-secrets
        args: ['--baseline', '.secrets.baseline']

CI Pipeline Secrets Scanning

# GitLeaks in GitHub Actions
- name: GitLeaks Scan
  uses: gitleaks/gitleaks-action@v2
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

5. DAST - Dynamic Application Security Testing

DAST tests running applications by simulating real attacks.

OWASP ZAP in CI/CD

DAST with OWASP ZAP
- name: ZAP Baseline Scan
  uses: zaproxy/action-baseline@v0.10.0
  with:
    target: 'https://staging.myapp.com'
    rules_file_name: '.zap/rules.tsv'
    issue_title: 'ZAP Security Scan Results'

When to Use DAST

  • Staging environment - After deployment to staging
  • Nightly builds - Full scan on main branch
  • Pre-release - Before production deployment

6. Container Security

Scan container images for vulnerabilities before deployment.

Trivy Container Scanning

Container Scan with Trivy
- name: Build Image
  run: docker build -t myapp:${{ github.sha }} .

- name: Trivy Scan
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: 'myapp:${{ github.sha }}'
    format: 'sarif'
    exit-code: '1'
    severity: 'CRITICAL,HIGH'

7. Infrastructure as Code Security

Scan Terraform, CloudFormation, and Kubernetes manifests for misconfigurations.

Checkov IaC Scanning

IaC Security with Checkov
- name: Checkov Scan
  uses: bridgecrewio/checkov-action@master
  with:
    directory: terraform/
    framework: terraform
    soft_fail: false
    skip_check: CKV_AWS_18,CKV_AWS_21

8. Complete Pipeline Example

Full DevSecOps Pipeline

This example combines all security tools into a comprehensive CI/CD workflow.

Complete .github/workflows/devsecops.yml
name: DevSecOps Pipeline
on: [push, pull_request]

jobs:
  secrets-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2

  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: returntocorp/semgrep-action@v1
        with:
          config: p/security-audit

  sca:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}

  container-scan:
    needs: [sast, sca]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t app:${{ github.sha }} .
      - uses: aquasecurity/trivy-action@master
        with:
          image-ref: 'app:${{ github.sha }}'
          exit-code: '1'
          severity: 'CRITICAL,HIGH'

  dast:
    needs: [container-scan]
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: zaproxy/action-baseline@v0.10.0
        with:
          target: 'https://staging.example.com'
Key Takeaways
  • Start with secrets scanning and SAST - they're fast and catch common issues
  • Add SCA next - dependency vulnerabilities are a major risk
  • Container scanning before any deployment
  • DAST on staging/pre-prod - not every commit (too slow)
  • Don't fail builds on every finding - prioritize critical/high first
