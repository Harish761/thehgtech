1. What is MEDUSA?

MEDUSA is a comprehensive Static Application Security Testing (SAST) tool built for the AI era. Created by Pantheon Security, it combines traditional code security scanning with cutting-edge AI agent security rules.

Unlike legacy SAST tools that focus solely on code vulnerabilities, MEDUSA recognizes that modern development involves AI assistants, LLM integrations, and agentic workflows - each introducing new attack vectors that traditional scanners miss.

2. Installation & Quick Start

System Requirements

  • Python 3.8 or higher
  • Windows, macOS, or Linux
  • 2GB RAM minimum (4GB recommended)

Installation

macOS/Linux (Recommended - Virtual Environment)

# Create and activate virtual environment
python3 -m venv medusa-env
source medusa-env/bin/activate

# Install MEDUSA
pip install medusa-security

# Verify installation
medusa --version

Windows

# Create and activate virtual environment
py -m venv medusa-env
medusa-env\Scripts\activate

# Install MEDUSA
pip install medusa-security

# Verify installation
medusa --version
Tip: For Windows, if the medusa command is not found, use py -m medusa instead.

5-Minute Setup

# 1. Initialize in your project
cd your-project
medusa init

# 2. Install security tools (auto-detected for your platform)
medusa install --all

# 3. Run your first scan
medusa scan .

Intelligent False Positive Reduction

MEDUSA's FP filter uses context-aware analysis to reduce noise by 40-60%:

  • Security wrapper detection: Recognizes SecureString, EncryptedCredential patterns
  • Docstring/comment exclusion: Ignores security-related documentation
  • Context-aware class analysis: Understands if code is in test files
  • Test/mock file identification: Reduces alerts for test fixtures
  • Known-safe pattern database: Curated list of false positive patterns

CVE Detection

MEDUSA actively detects known critical CVEs:

CVE-2025-55182 "React2Shell": CVSS 10.0 RCE vulnerability affecting React Server Components (19.0.0-19.2.0) and Next.js App Router (15.0.0-15.0.4). MEDUSA scans package.json, package-lock.json, yarn.lock, and pnpm-lock.yaml.
CVE-2025-6514: MCP-remote RCE vulnerability affecting Model Context Protocol servers. MEDUSA detects vulnerable MCP configurations.

6. Real-World Usage Examples

Scanning a Python Project

# Full scan with all reports
medusa scan ./my-python-app --format all

# Quick scan (changed files only)
medusa scan ./my-python-app --quick

# Scan with 8 workers for faster processing
medusa scan ./my-python-app --workers 8

Scanning a Node.js/React Project

# Check for React2Shell CVE
medusa scan ./my-react-app

# Output includes:
# üö® CRITICAL: CVE-2025-55182 React2Shell detected
#    package.json: react@19.1.0 (vulnerable: 19.0.0-19.2.0)

Scanning AI Agent Configurations

# Scan only AI-related files
medusa scan . --ai-only

# Scan MCP server configuration
medusa scan ./mcp-servers/

Fail on High Severity (CI/CD)

# Fail the build if HIGH or CRITICAL issues found
medusa scan . --fail-on high

# Exit codes:
# 0 = No issues at or above threshold
# 1 = Issues found at or above threshold
# 2 = Scan error

8. CI/CD Integration

GitHub Actions Example

name: Security Scan

on: [push, pull_request]

jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          
      - name: Install MEDUSA
        run: pip install medusa-security
        
      - name: Initialize MEDUSA
        run: medusa init
        
      - name: Install scanners
        run: medusa install --all --yes
        
      - name: Run security scan
        run: medusa scan . --format sarif --fail-on high
        
      - name: Upload SARIF to GitHub
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: .medusa/reports/*.sarif

GitLab CI Example

security_scan:
  stage: test
  image: python:3.11
  script:
    - pip install medusa-security
    - medusa init
    - medusa install --all --yes
    - medusa scan . --format json --fail-on high
  artifacts:
    reports:
      sast: .medusa/reports/*.json

10. Best Practices

When to Run MEDUSA

  • Pre-commit: Quick scan on changed files
  • Pull Request: Full scan with --fail-on high
  • Main branch: Comprehensive scan with all reports
  • Weekly: Full audit scan with AI-only focus

Configuration Best Practices

# .medusa.yml
version: "1.0"

scan:
  workers: 6
  cache: true
  fail_on: high
  
exclude:
  - "node_modules/"
  - "venv/"
  - "*.test.py"
  - "**/__tests__/**"

ai_security:
  enabled: true
  strict_mode: true
  
reports:
  formats:
    - json
    - sarif
  output_dir: ".medusa/reports"

Handling Findings

Prioritization Framework:
  1. CRITICAL: Fix immediately - CVEs, prompt injection in production
  2. HIGH: Fix before merge - Tool poisoning, RAG vulnerabilities
  3. MEDIUM: Plan for next sprint - Configuration issues
  4. LOW: Track and address - Style/best practice issues

Team Workflow

  1. Add MEDUSA to CI/CD pipeline
  2. Set --fail-on high for pull requests
  3. Review findings in weekly security triage
  4. Track false positives and update .medusa.yml exclusions
  5. Run --ai-only scans when updating AI configurations
