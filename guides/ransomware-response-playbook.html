Critical: Time is Everything

In a ransomware incident, every minute counts. This playbook provides a structured, step-by-step response framework to minimize damage, contain the threat, and restore operations. Print this guide and keep it accessible for your incident response team.

Executive Summary

Ransomware attacks have evolved from opportunistic malware to sophisticated, targeted campaigns orchestrated by organized cybercrime groups. Modern ransomware employs double extortion (encryption + data theft), lateral movement, and credential harvesting to maximize impact and ransom demands.

This playbook covers the complete incident response lifecycle: Preparation, Detection, Containment, Eradication, Recovery, and Post-Incident Analysis.

Phase 1: Immediate Response (0-1 Hour)

Golden Hour: First 60 Minutes

The first hour determines the scope of damage. Rapid containment can prevent ransomware from spreading to backups, domain controllers, and critical systems.

Step 1: Activate Incident Response Team

  • Notify IR Team Lead: Immediately alert the designated incident commander
  • Assemble Core Team: Security analysts, network engineers, system administrators, legal counsel
  • Establish War Room: Set up a dedicated communication channel (Slack, Teams, or conference bridge)
  • Document Everything: Start an incident log with timestamps for all actions taken

Step 2: Isolate Affected Systems

Do NOT power off systems immediately ‚Äî this can destroy volatile memory evidence and encryption keys. Instead:

  • Network Isolation: Disconnect affected systems from the network (disable Wi-Fi, unplug Ethernet)
  • Disable Remote Access: Block VPN, RDP, SSH access to prevent lateral movement
  • Segment Networks: Use firewall rules to isolate infected VLANs/subnets
  • Preserve Evidence: Take memory dumps and disk images before any remediation
# Emergency network isolation (Linux)
sudo iptables -A INPUT -j DROP
sudo iptables -A OUTPUT -j DROP

# Emergency network isolation (Windows PowerShell)
Disable-NetAdapter -Name "Ethernet" -Confirm:$false
Disable-NetAdapter -Name "Wi-Fi" -Confirm:$false

Step 3: Identify Ransomware Variant

Identifying the ransomware family helps determine:

  • Whether decryption tools exist (e.g., from No More Ransom Project)
  • Known TTPs (Tactics, Techniques, and Procedures)
  • Threat actor attribution and typical ransom demands

Identification Methods:

  • Check ransom note file name and extension (e.g., .lockbit, .conti)
  • Upload ransom note to ID Ransomware
  • Analyze file encryption patterns and extensions
  • Review process names and registry modifications

Phase 2: Containment (1-4 Hours)

Step 4: Prevent Lateral Movement

  • Disable Domain Admin Accounts: Temporarily disable high-privilege accounts
  • Reset Credentials: Force password resets for all service accounts and admin accounts
  • Block C2 Communication: Identify and block command-and-control (C2) IP addresses and domains
  • Monitor Network Traffic: Use IDS/IPS to detect anomalous outbound connections

Step 5: Protect Backups

Critical: Backup Protection

Modern ransomware actively targets backups. If backups are encrypted, recovery becomes impossible.

  • Disconnect Backup Systems: Immediately isolate backup servers and NAS devices
  • Verify Backup Integrity: Check that recent backups are unencrypted and accessible
  • Enable Immutable Backups: If using cloud backups, enable object lock/versioning
  • Test Restore Capability: Verify you can restore from backups before proceeding

Phase 3: Eradication (4-24 Hours)

Step 7: Remove Malware

  • Boot into Safe Mode: Prevents ransomware from executing on startup
  • Run Anti-Malware Scans: Use multiple tools (Malwarebytes, ESET, Kaspersky)
  • Delete Persistence Mechanisms: Remove registry keys, scheduled tasks, startup entries
  • Rebuild Compromised Systems: For critical systems, rebuild from clean images
# Check for scheduled tasks (Windows PowerShell)
Get-ScheduledTask | Where-Object {$_.TaskPath -notlike "\Microsoft*"}

# Check for suspicious registry run keys
reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run
reg query HKCU\Software\Microsoft\Windows\CurrentVersion\Run

Step 8: Patch Vulnerabilities

Ransomware often exploits known vulnerabilities. Ensure all systems are patched:

  • Apply Windows security updates (especially for SMB, RDP vulnerabilities)
  • Update VPN appliances (Fortinet, Pulse Secure, Palo Alto)
  • Patch web applications and CMS platforms
  • Update endpoint protection software

Phase 4: Recovery (24-72 Hours)

Step 9: Restore from Backups

1. Verify Backup Integrity

Test restore on isolated systems first. Verify data is not corrupted or encrypted.

2. Restore Critical Systems First

Prioritize: Domain controllers ‚Üí File servers ‚Üí Databases ‚Üí Applications ‚Üí Workstations

3. Validate Restored Data

Check file integrity, application functionality, and database consistency.

4. Monitor for Re-Infection

Use EDR tools to detect any signs of malware re-execution.

Step 10: Rebuild Compromised Systems

For systems that cannot be reliably cleaned, perform a full rebuild:

  • Reformat drives and reinstall OS from trusted media
  • Apply all security patches before connecting to network
  • Restore data from verified clean backups
  • Reinstall applications from official sources

Phase 5: Post-Incident Analysis

Step 11: Conduct Forensic Investigation

Forensic Analysis Goals
  • Determine initial infection vector (phishing, RDP brute force, vulnerability exploit)
  • Identify dwell time (how long attackers had access before deploying ransomware)
  • Map lateral movement and compromised accounts
  • Assess data exfiltration (for double-extortion scenarios)

Step 12: Regulatory and Legal Obligations

  • Notify Law Enforcement: Report to FBI IC3 or local cybercrime unit
  • Regulatory Reporting: Comply with GDPR, HIPAA, or other data breach notification laws
  • Cyber Insurance: File claim with cyber insurance provider
  • Customer Notification: If PII was compromised, notify affected individuals

Step 13: Implement Lessons Learned

  • Update incident response plan based on gaps identified
  • Enhance monitoring and detection capabilities
  • Conduct tabletop exercises to test new procedures
  • Implement additional security controls (MFA, network segmentation, EDR)

Ransom Payment Decision Framework

To Pay or Not to Pay?

The FBI and CISA recommend NOT paying ransoms, as it funds criminal operations and does not guarantee data recovery. However, some organizations face impossible choices.

Factor Consider Payment If... Avoid Payment If...
Backups No viable backups exist Clean backups are available
Business Impact Downtime threatens survival Operations can continue
Data Sensitivity Exfiltrated data is highly sensitive No data was stolen
Decryption Tools No free decryptor available Free decryptor exists
Legal Constraints Payment is legally permissible Sanctioned threat actor (OFAC)

If payment is considered: Engage a professional ransomware negotiation firm. They can reduce ransom demands by 50-70% and verify decryption key validity before payment.

Prevention: Building Ransomware Resilience

Technical Controls

  • Endpoint Detection and Response (EDR): Deploy behavioral detection (CrowdStrike, SentinelOne, Microsoft Defender)
  • Email Security: Use advanced email filtering to block phishing (Proofpoint, Mimecast)
  • Network Segmentation: Isolate critical systems from general network
  • MFA Everywhere: Enforce multi-factor authentication on all remote access
  • Disable RDP: Or restrict to VPN-only access with MFA
  • Application Whitelisting: Prevent unauthorized executables from running

Backup Strategy (3-2-1-1-0 Rule)

  • 3 copies of data
  • 2 different media types
  • 1 copy offsite
  • 1 copy offline (air-gapped)
  • 0 errors in backup verification

User Awareness Training

  • Conduct quarterly phishing simulations
  • Train users to recognize social engineering tactics
  • Establish clear reporting procedures for suspicious emails

Conclusion

Ransomware incidents are no longer a matter of "if" but "when." Organizations that prepare in advance ‚Äî with robust backups, tested incident response plans, and layered security controls ‚Äî can recover quickly and minimize damage. This playbook provides a battle-tested framework, but it must be adapted to your organization's specific environment and regularly updated as threats evolve.
